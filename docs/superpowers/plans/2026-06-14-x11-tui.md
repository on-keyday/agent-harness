# X11 forwarding — TUI support (follow-up)

> Closes the spec gap: `docs/superpowers/specs/2026-06-14-x11-forwarding.md` said "CLI/TUI"; the first round shipped CLI only. This adds the TUI path.

**Goal:** `harness-tui` can start an X11-forwarded interactive session (`session new --x11 [--x11-display N]` in the TUI cmdline), mirroring the CLI.

**Architecture:** Extract the reusable X11 prep+open into an exported `cli.OpenInteractiveX11` (returns the PTY stream, task id, the `-R` `RemoteForwardSpec`, and a `warn` string for the no-auth case — no stderr printing inside, so the TUI's alt-screen isn't corrupted). The CLI's `RunInteractiveX11` refactors onto it. The TUI runs the forward in a background goroutine (cancel tied to session end) and foregrounds the PTY via its existing `tea.Exec(interactiveExec{stream})` path; the forward's logf uses the existing buffered `forwardStatusLogf` (because `program.Send` blocks during `tea.Exec`).

**Key files:** `cli/x11.go`, `cli/open_interactive_native.go` (none — openInteractive already takes x11), `tui/cmdline.go`, `tui/app.go`, `tui/interactive.go`.

```go
// OpenInteractiveX11 opens an X11-forwarded interactive session and returns the
// PTY stream, task id, the -R RemoteForwardSpec the caller must run for the
// session's lifetime, and a non-empty warn string when forwarding without
// authentication (no cookie). The caller decides how to surface warn (CLI →
// stderr; TUI → status line).
func (c *Client) OpenInteractiveX11(ctx context.Context, repo string, sel protocol.RunnerSelector, extraArgs []string, resumeTaskID string, displayN int) (*agentexec.CommandExecutionStream, string, RemoteForwardSpec, string, error) {
	display := os.Getenv("DISPLAY")
	network, host, port, err := localXServerDialSpec(display)
	if err != nil {
		return nil, "", RemoteForwardSpec{}, "", err
	}
	var warn string
	cookie, err := localX11Cookie(display)
	if err != nil {
		warn = fmt.Sprintf("no cookie for %s (%v); forwarding WITHOUT authentication — your X server must accept unauthenticated connections", display, err)
		cookie = nil
	}
	stream, taskIDHex, err := c.openInteractive(ctx, repo, sel, extraArgs, resumeTaskID, true, &X11Request{Display: displayN, Cookie: cookie})
	if err != nil {
		return nil, taskIDHex, RemoteForwardSpec{}, warn, err
	}
	sp := RemoteForwardSpec{BindAddr: "127.0.0.1", RunnerPort: 6000 + displayN, DialNetwork: network, DialHost: host, DialPort: port}
	return stream, taskIDHex, sp, warn, nil
}
```
- [ ] Refactor `RunInteractiveX11` to call it (preserve CLI behavior incl. the stderr warning + banner):
```go
func (c *Client) RunInteractiveX11(ctx context.Context, repo string, sel protocol.RunnerSelector, extraArgs []string, resumeTaskID string, displayN int) (string, error) {
	stream, taskIDHex, sp, warn, err := c.OpenInteractiveX11(ctx, repo, sel, extraArgs, resumeTaskID, displayN)
	if err != nil {
		return taskIDHex, err
	}
	defer stream.Close()
	if warn != "" {
		fmt.Fprintln(os.Stderr, "x11: "+warn)
	}
	fctx, cancel := context.WithCancel(ctx)
	defer cancel()
	go func() {
		logf := func(s string) { fmt.Fprintln(os.Stderr, "x11: "+s) }
		if err := RunRemoteForward(fctx, c, taskIDHex, []RemoteForwardSpec{sp}, logf); err != nil {
			fmt.Fprintln(os.Stderr, "x11 forward: "+err.Error())
		}
	}()
	fmt.Fprintf(os.Stderr, "harness-cli: X11 session %s (remote DISPLAY=127.0.0.1:%d -> local %s; Ctrl+] detach, Ctrl+D/exit ends)\n", taskIDHex, displayN, os.Getenv("DISPLAY"))
	if err := stream.RemoteShell(); err != nil {
		return taskIDHex, err
	}
	return taskIDHex, nil
}
```
